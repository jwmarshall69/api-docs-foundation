# Weekly Success Criteria

This file defines completion criteria for each training week to ensure
consistent progress and measurable outcomes.

---

## Week 1 – Repository Foundation
**Success Criteria**
- Repository created with agreed folder structure
- Templates added but not over-customized
- Initial commit completed
- Repo feels organized and intentional

---

## Week 2 – Core Documentation Pages
**Success Criteria**
- Authentication documentation completed
- Error handling documented with example
- One endpoint fully documented
- Docs are usable without verbal explanation

---

## Week 3 – Tool Fluency Lock-In
**Success Criteria**
- OpenAPI spec aligned with docs
- Postman collection matches documented endpoints
- Clean commit history with meaningful messages
